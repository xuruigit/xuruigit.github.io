使用Hexo + GitHub Pages搭建博客

发表于 2018-01-03 | 更新于 2018-11-14

万事开头难，第一篇就记录下这个博客搭建过程吧。

为什么选择 Hexo ？

  1. 支持markdown（最好用的markdown编辑器是macdown，不过不支持归档分类，现在转用MWeb了）
  2. 优秀主题多，本人颜控，不好看的一律pass
  3. 安装、部署、发布流程简单，一键部署到GitHub Pages上
  4. 支持归档、分类、目录
  5. 速度快

为什么选择托管在 GitHub Pages 上？

  1. 稳定、免费
  2. 逼格高那么一丢丢
  3. 也有缺陷，速度慢，访问不稳定，替代方案：托管在国内的coding上，但是万一哪天给你插个广告也是受不了

搭建流水账

安装配置Hexo

看官方文档就可以了，中文版：安装、配置

主题设置

主题其实就是我们生成博客时使用的样式，官方主题：Themes， 我使用的主题是Maupassant，简洁风深得我心。

安装主题

将主题代码clone到hexo下的themes文件夹内

启用主题

将hexo目录下的_config.yml配置文件中的theme属性改为：maupassant

更新主题

默认端口4000，打开http://localhost:4000/ ，主题就更新了。

GitHub Pages设置

GitHub Pages 原本是用来介绍托管在GitHub上的项目，有免费、稳定的优点，也很适合用于搭建博客。
有两点需要注意一下：

流程网上很多就不搬运了，可以参考：一步步在GitHub上创建博客主页

之后 http://xuruigit.github.io 就是我的github个人主页了，下一步就是让这个主页显示我们的博客内容。

部署Hexo到GitHub Pages

这一步的目的是把博客内容部署到github上，利用GitHub Pages显示出来
整理一下原理：

  1. Hexo 通过 hexo g 命令，将我们写的.md文件生成html静态文件到public文件夹下
  2. Hexo 通过 hexo d 命令，将public下的文件deploy到github的xuruigit.github.io仓库的master分支下
  3. 当我们请求http://xuruigit.github.io 时，github将请求转发到master分支下的静态html上

部署流程

安装扩展：

修改配置文件_config.yml（冒号后面必须有空格）

执行deploy

还是很方便的。

绑定域名

要逼格，必须有自己的域名啊

购买域名

我的域名是在阿里云上购买的: xurui.pro ，5年费用￥371

配置域名解析

域名解析的目的是为了将咱们购买的域名的请求转发到github上

  1. 创建2条A类型的记录解析到192.30.252.154、192.30.252.153，这2个ip是github提供的地址
  2. 创建CNAME记录解析到xuruigit.github.io.（为什么io后面要有个.？）
  3. github配置CNAME
    进入source目录下，添加CNAME文件，填入购买的域名：xurui.pro

构建部署到github上

等待域名解析生效，访问 www.xurui.pro 就可以啦
